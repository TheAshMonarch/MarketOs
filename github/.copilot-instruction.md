# MarketOS Frontend — Project Rules

## Stack (do not substitute or add alternatives)
Next.js 15 App Router, React 19, TypeScript, Tailwind CSS, shadcn/ui, React Hook Form, Zod.
Backend is separate (Next.js API routes, serverless) — never write backend logic, only fetch calls to /api/*.

## Design rules — strict
- Do NOT use: cream background + terracotta/clay accent (#D97757-ish), near-black + acid-green/vermillion, or broadsheet-style hairline rules with zero border-radius. These are banned defaults.
- Palette: warm neutral base (not stark white, not cream-default) + one confident accent from indigo/adire-dye blue, market-tarp green, or deep ochre/gold.
- Typography: a display face with character for headers, clean legible body face for numbers/data. No generic Inter-for-everything.
- No numbered markers (01/02/03) unless content is a genuine sequence.
- Minimal animation. Generous whitespace. Mobile-first, responsive down to small screens. Visible focus states.

## Scope — exactly 6 pages, nothing else
1. Register/Login (with phone OTP step)
2. Passport (owner view/edit) — business name, owner name, market, shop number, category, photo, description, own stats
3. Passport (public view) — /passport/[businessId], read-only, no login required
4. Smart Sale Entry — free-text textarea → POST /api/ai/parse → editable confirmation list (product, quantity, unit — editable/removable, plus add-manually) → Confirm & Save → POST /api/sales
5. Dashboard — 4 stat cards (Revenue, Sales Today, Transactions, Products in Stock), 1 line chart (Weekly Sales, Recharts), Recent Activity list
6. Receipt/QR Verify — receipt + QR after sale; /receipt/[receiptId] public page showing verified/not-verified, amount, seller, timestamp

## Explicitly out of scope — never generate these
Forgot-password flow, product edit/delete/search, low-stock alerts, settings page, any AI-generated charts beyond plain numbers from GET /api/dashboard.

## API contract (already fixed, do not invent new endpoints)
POST /api/auth/register, /api/auth/login, /api/auth/verify-otp
GET/POST /api/business, GET /api/business/:id
POST /api/ai/parse, POST /api/sales
GET /api/dashboard
GET /api/receipt/:id
